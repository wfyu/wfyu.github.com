老实说，在第一次见到vim这个文本编辑器时，可能和大家的心理是以样的，这么原始的东西，何以被大虾们说得神乎其神呢，实在百思不得其解，然后呢，我就放弃了， 直到一次偶然间，发现了一位高人的博客，算是我的vim启蒙老师吧，我的博客就是用 vim写出来的，当然还得借助与一个插件即：vimwiki。不然写出来的网页可就没那么绚丽诺。下面就教大家如何用vimwiki来写一个网页吧!

安装vim

本实例在ubuntu10.04上进行，其它Linux版本类似，打开终端输入 

 sudo apt-get install vim

Linux下Vim的配置文件名是.vimrc，位于你自己的家目录下，如:/home/ruchee/.vimrc。该文件是隐藏文件，需要使用ls -a才能看到，如果没有也可以自己手动新建一个,这里我附上一个配置文件的下载链接：../../../doc/vimrc.txt 请将vimrc这个文件拷贝到家目录，并重命名为.vimrc .

安装vimwiki

  1. 由于ubuntu软件中心没有收录这个软件，所以需要我们自己去网页上下载，这里提供下载链接[http://code.google.com/p/vimwiki/]
  2. 使用vim 打开vimwiki-xxxx.vba 这个文件
  3. 在normal 模式下执行命令，:so %，不多久就会自动安装完成
  4. 修改.vimrc配置文件里面的如下语句 
" 打开语法高亮
syntax enable
syntax on
" 打开对文件类型插件的支持
filetype indent on
filetype plugin on
filetype plugin indent on
" ######### VimWiki 写作助手 ######### "
" 使用鼠标映射
let g:vimwiki_use_mouse = 1

" 不要将驼峰式词组作为 Wiki 词条
let g:vimwiki_camel_case = 0

let g:vimwiki_list = [{
\ 'path': 'D:/mysite/wiki',
\ 'path_html': 'D:/mysite/html/',
\ 'auto_export': 1,
}]

这个配置文件是针对我电脑的环境的，每个人需要根据自己的路径进行修改，不过很简单，我来一一解说：

  • g:vimrc_author='oldboys'
  • g:vimrcrc_email='yezi_Linux@163.com'
  • g:vimrc_homepage='http://oldboys.sinaapp.com' 以上3项根据个人信息填写，这个不是重要的，只是作下说明而已，下面的可就重要了
  • path: 指定你存放vimwiki源代码的地方
  • path_html: 指定编译得到的html文件生成的地方
  • auto_export：每当保存wiki源文件时自动编译，关闭设为0
  • html_header：指定编译时使用的头部模板
  • html_footer：指定编译时使用的尾部模板

现在再次打开 Vim，直接敲入 \ww，这样就进入了你的wiki首页，或者在终端创建以 .wiki 为后缀名的文件时，也会自动进入vimwiki 的编辑环境。

基本语法

  1. == 二级标题 ==：二级标题，我一般用来标识文章标题（如果行首空一个空格则会居中，适用于所有标题标签）
  2. === 三级标题 ===：三级标题，我一般用来标识文章子标题
  3. ====== 六级标题 ====：六级标签，我用来标记时间（标题标签共6个，对应于等于号的个数，六级标签字体最小，一级标签字体最大，本博客首页那些粉红色标题就是用的一级标签）
  4. [[链接地址|链接标题]]：这主要用来链接本地文件，如本页首部的[首页] - [Linux] - [Vim]，其对应写法为[[../../../index|首页]] - [[../index|Linux]] - [[index|Vim]]
  5. [链接地址 链接标题]：这主要用来贴链接，如我想贴个Google首页的链接可以这样写 [http://www.google.com Google]，不过这个标签我基本上没怎么用过，因为都没几个外部链接
  6. [[转向地址|图片地址]]：本博简介页面的头像是这样写的[[index.html|images/ruchee.jpg]]，这样就会显示对应的图片，而点击图片就会转到首页。如果不需要转向，直接写[[图片地址]]即可
  7. ----：四条小横线相当于< hr >标签，会出现一条本博客到处可见的横线
  8. {{{ 程序代码或不需要编译的内容 }}}：三个大括号用来书写程序代码或其他不想被编译的内容。注意，为了防止这些演示标签生效，我加上了空格，实际使用时不要留空格
  9. 两个反斜线（这儿不便写演示代码，反斜线一般位于数字键1的左边）：相当于< code >与< /code >标签，显示效果就是本页随处可见的绿色荧光字
  10. 以上就是我用的最多的VimWiki标签，更多帮助请输入：help vimwiki 寻求帮助
